# ClawFeeder Agent - Technical Design

**Version:** 1.2
**Date:** 2026-04-09
**Module:** Headless Worker Agent

---

## 1. Overview

**Purpose:** Headless worker that polls backend for keep-alive tasks, decrypts payloads locally, executes HTTP requests, and persists decrypted cookies to local filesystem.

**Core Principle:** "Client executes (Worker)." The agent polls for tasks, decrypts locally using the master key, executes HTTP requests from the user's real local IP, reports results, and persists cookies for other tools (AI agents, crawlers).

## 4. Config Design

Two-tier separation: build-time config baked into the binary, user config in `~/.clawfeeder/config.yaml`.

### 4.1 Build-time Config (`config.build.yaml`)

Set by the deployer, consumed by `build.sh`. **Never distributed to users.**

```yaml
api:
  base_url: "https://clawfeeder.example.com"
  heartbeat_interval: 60
```

`build.sh` reads this file, generates `src/_build_config.py`, and runs PyInstaller. The resulting binary has `base_url` and `heartbeat_interval` compiled in.

### 4.2 User Config (`~/.clawfeeder/config.yaml`)

Generated by `install.sh`. Only contains user-specific settings.

```yaml
# User config вҖ” edit this file to update your personal settings.
# Changes take effect on agent restart.

auth:
  api_key: "cf_agt_..."

storage:
  data_dir: "~/.clawfeeder/data"
  expired_dir: "~/.clawfeeder/data/expired"

device:
  device_id: "auto-generated-uuid"
  device_name: "My MacBook Pro"

master_key: "your_master_password"
```

### 4.3 Config Resolution (src/config.py)

| Value | Source | Fallback |
|-------|--------|---------|
| `base_url` | `_build_config.BASE_URL` (baked) | `config.yaml api.base_url` (dev only) |
| `heartbeat_interval` | `_build_config.HEARTBEAT_INTERVAL` (baked) | `config.yaml api.heartbeat_interval` |
| `api_key` | `config.yaml auth.api_key` | вҖ” |
| `master_key` | `config.yaml master_key` | `MASTER_KEY` env var |
| `device_id` | `config.yaml device.device_id` | вҖ” |
| `device_name` | `config.yaml device.device_name` | вҖ” |

## 6. Data Flow

### 6.1 Heartbeat Loop

```
Every ~60 seconds:
        в”Ӯ
        в–ј
Agent: POST /api/v1/agent/heartbeat { device_id, device_name }
        в”Ӯ
        в–ј
Server: Returns available task IDs + encrypted payloads
        в”Ӯ
        в–ј
For each task:
        в”Ӯ
        в”ңв”Җв”Җ Agent decrypts encrypted_data in memory
        в”Ӯ       (master key from config.yaml)
        в”Ӯ
        в”ңв”Җв”Җ Agent executes HTTP request from LOCAL IP
        в”Ӯ       в”Ӯ
        в”Ӯ       в”ңв”Җв”Җв”Җ Success (2xx)?
        в”Ӯ       в”Ӯ       в””в”Җв”Җв”Җ PATCH /cookies/{id}/status { status: 'active' }
        в”Ӯ       в”Ӯ
        в”Ӯ       в”ңв”Җв”Җв”Җ Auth failure (401/403)?
        в”Ӯ       в”Ӯ       в””в”Җв”Җв”Җ PATCH /cookies/{id}/status { status: 'expired' }
        в”Ӯ       в”Ӯ
        в”Ӯ       в””в”Җв”Җв”Җ Server error (5xx)?
        в”Ӯ               в””в”Җв”Җв”Җ Retry 3x with exponential backoff
        в”Ӯ
        в”ңв”Җв”Җ Agent writes decrypted cookies to local files
        в”Ӯ
        в””в”Җв”Җ Agent releases lock (via PATCH status)

After every 5 heartbeat cycles вҶ’ Local File GC:
        в”Ӯ
        в–ј
GET /api/v1/cookies (active only)
        в”Ӯ
        в–ј
Compare cloud domains vs local files
  - Domain not in cloud вҶ’ DELETE local file
  - Domain expired вҶ’ MOVE to expired/
```

## 7. Decryption Flow

```
Master Password (from config.yaml)
      в”Ӯ
      в”ңв”Җв”Җ PBKDF2 (100,000 iterations, SHA-256)
      в”Ӯ   Salt: "clawfeeder-salt-v1" (fixed, must match extension)
      в”Ӯ
      в–ј
   256-bit AES Key
      в”Ӯ
      в–ј
   AES-256-GCM Decrypt
```

**Critical:** `SALT` and PBKDF2 parameters must stay in sync with `lib/crypto.js` in the extension.

## 8. API Integration

### 8.1 Authentication

All requests use Bearer token:
```
Authorization: Bearer cf_agt_...
```

### 8.2 Endpoints Used

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/agent/heartbeat` | Poll for tasks, update device last_seen |
| GET | `/api/v1/cookies` | List cookies (for GC) |
| PATCH | `/api/v1/cookies/{id}/status` | Report keep-alive result |

## 12. Security Considerations

- API key (`cf_agt_...`) authenticates to backend вҖ” distinct from master key
- Master key used only for local E2EE decryption, never sent to backend
- Decrypted cookies written to local filesystem вҖ” protect `~/.clawfeeder/data/`
- Build-time config (`base_url`) baked into binary, not user-editable
- All backend communication over HTTPS
